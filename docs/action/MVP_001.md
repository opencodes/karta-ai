# 🛠️ Sprint 1: Karta-Do (Task Creation)

## 1. Feature Goal
Allow users to create tasks using "Natural Language Input" (NLI). Instead of clicking 5 buttons, the user should type a single sentence.

## 2. The User Flow (UX)
1. **Input:** User types: "Remind me to pay the electricity bill tomorrow at 4 PM."
2. **Karta-AI Processing:** - *Entity Extraction:* Task = "Pay bill", Category = "Finance", Time = "Tomorrow 4 PM".
3. **Execution:** Karta creates the task AND links it to the Finance module.

## 3. Technical Requirements (MVP)
- **Input Field:** A central command bar (Cmd+K interface).
- **Backend Logic:** A simple parser (using an LLM or Regex) to extract Date, Time, and Intent.
- **Database Schema:**
  - `id`: Unique ID
  - `task_name`: String
  - `category`: [Finance, Personal, Work, Contact]
  - `due_date`: DateTime
  - `is_completed`: Boolean

## 4. Definition of Done (DoD)
- User can type a task and hit Enter.
- Task appears in a "Today" or "Upcoming" list.
- Task shows the correct "Category" icon automatically.


Since we are starting with **Karta-Do** (Task Creation), the goal is to build a "Zero-Friction" interface. In Karta, a task isn't just a string of text; it’s an **object** with intent.

### 2. The Logic: Basic Natural Language Parser

For the MVP, you don't need a heavy LLM for every keystroke. You can use a simple Javascript function to "catch" keywords and dates.

**Karta-Do Logic Example:**

```javascript
function parseKartaInput(input) {
    const now = new Date();
    let task = {
        title: input,
        category: "General",
        dueDate: new Date()
    };

    // Keyword Detection (The "Agent" Thinking)
    if (input.toLowerCase().includes("pay") || input.toLowerCase().includes("bill")) {
        task.category = "Finance";
    }
    
    // Simple Date Detection
    if (input.toLowerCase().includes("tomorrow")) {
        task.dueDate.setDate(now.getDate() + 1);
    }

    return task;
}

```
